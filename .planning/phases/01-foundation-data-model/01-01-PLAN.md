  <action>
  Run `pnpm create-next-app@latest .` in the working directory with these EXACT flags:
  
  ```bash
  pnpm create-next-app@latest . \
    --typescript \
    --tailwind \
    --app \
    --src-dir \
    --import-alias "@/*" \
    --use-pnpm \
    --eslint \
    --no-turbopack
  ```
  
  - `--typescript`: Enable TypeScript (non-negotiable)
  - `--tailwind`: Enable Tailwind CSS v4 (project standard)
  - `--app`: Use App Router (required — Pages Router is deprecated in Next.js 16)
  - `--src-dir`: Place source in `src/` directory
  - `--import-alias "@/*"`: Use `@/` path alias for clean imports
  - `--use-pnpm`: Use pnpm as package manager (per D-01)
  - `--eslint`: Enable ESLint config
  - `--no-turbopack`: Use Webpack (Turbopack creates issues with some libraries; Webpack is stable and well-tested)
  
  If the working directory is not empty, create-next-app will prompt to overwrite. Answer "yes" to all prompts (use `--yes` flag if available, otherwise pipe "y").
  
  **DO NOT** init a git repo or commit — that happens in Plan 04 (GitHub + CI/CD).
  
  After the command completes, verify:
  - `ls src/app/page.tsx` shows the main page
  - `ls src/app/layout.tsx` shows the root layout
  - `ls src/app/globals.css` shows global styles
  - `ls package.json` shows the project manifest
  </action>

<task type="auto">
  <name>task 3: create Drizzle config, env templates, and .gitignore entries</name>
  
  <files>drizzle.config.ts, .env.example, .env.local</files>
  
  <read_first>
  Read `package.json` to confirm project name and that drizzle-kit is installed as a devDependency.
  Read `next.config.ts` to confirm Next.js 16 project structure.
  </read_first>
  
  <action>
  Create three files:

  ### (1) drizzle.config.ts
  Write `drizzle.config.ts` at the project root:
  ```typescript
  import { defineConfig } from "drizzle-kit";

  export default defineConfig({
    schema: "./src/lib/db/schema.ts",
    out: "./src/lib/db/migrations",
    dialect: "sqlite",
    dbCredentials: {
      url: process.env.TURSO_DATABASE_URL ?? "file:./data/dev.db",
    },
  });
  ```
  
  - `dialect: "sqlite"` — Turso uses libSQL which is SQLite-compatible
  - `schema` points to where Plan 02 will create the schema file
  - `out` specifies where migration files will be generated
  - `url` uses `TURSO_DATABASE_URL` env var with a local fallback to `file:./data/dev.db` for development (per D-03: local SQLite dev, Turso remote prod)

  ### (2) .env.example
  Write `.env.example`:
  ```env
  # Turso Database (Production)
  TURSO_DATABASE_URL=libsql://your-database-name.turso.io
  TURSO_AUTH_TOKEN=your-turso-auth-token

  # Development (leave empty to use local SQLite file)
  # TURSO_DATABASE_URL=file:./data/dev.db

  # Next.js
  NEXT_PUBLIC_APP_URL=http://localhost:3000
  ```

  ### (3) .env.local
  Write `.env.local` for local development:
  ```env
  # Use local SQLite file for development (per D-03)
  TURSO_DATABASE_URL=file:./data/dev.db
  NEXT_PUBLIC_APP_URL=http://localhost:3000
  ```

  ### (4) Update .gitignore
  Check if `.gitignore` exists (created by create-next-app). Append these entries if not already present:
  ```
  # Local env files (secrets)
  .env.local
  .env.*.local
  
  # Local database files
  /data/
  *.db
  *.db-journal
  
  # Drizzle Kit meta
  drizzle-kit/**
  !drizzle-kit/migrator
  ```

  After creating these files, verify:
  - `pnpm build` still passes (config changes don't break compilation)
  - `ls data/` shows the directory exists (create it if not: `mkdir -p data/`)
  </action>

  - `.env.local` exists with `TURSO_DATABASE_URL=file:./data/dev.db`
