<task type="auto">
  <name>task 2: create GitHub Actions CI/CD workflow</name>
  
  <files>.github/workflows/ci.yml</files>
  
  <read_first>
  Read `package.json` to confirm the exact lint and type-check script commands.
  Read `pnpm-lock.yaml` or `package.json` to confirm pnpm is the package manager (D-01).
  </read_first>
  
  <action>
  Create the CI directory and workflow file:

  ```bash
  mkdir -p .github/workflows
  ```

  Write `.github/workflows/ci.yml`:
  ```yaml
  name: CI

  on:
    push:
      branches: [main]
    pull_request:
      branches: [main]

  jobs:
    validate:
      runs-on: ubuntu-latest

      steps:
        - uses: actions/checkout@v4

        - name: Setup pnpm
          uses: pnpm/action-setup@v4
          with:
            version: 11

        - name: Setup Node.js
          uses: actions/setup-node@v4
          with:
            node-version: 24
            cache: "pnpm"

        - name: Install dependencies
          run: pnpm install --frozen-lockfile

        - name: Run lint
          run: pnpm lint

        - name: Run type-check
          run: pnpm type-check
  ```

  **IMPORTANT:** Check if `package.json` has a `type-check` script. Next.js 16's `create-next-app` generates a `lint` script but may not generate `type-check`. If `type-check` is missing from `package.json`, add it:
  ```bash
  # Check if type-check script exists
  grep '"type-check"' package.json
  
  # If missing, add it to the scripts section
  ```
  
  Edit `package.json` to add the `type-check` script:
  ```json
  "scripts": {
    "dev": "next dev",
    "build": "next build",
    "start": "next start",
    "lint": "next lint",
    "type-check": "tsc --noEmit"
  }
  ```

  After creating the workflow file, verify it locally:
  ```bash
  # Test that lint and type-check work
  pnpm lint 2>&1 | tail -5
  pnpm type-check 2>&1 | tail -5
  ```
  
  Both commands MUST exit 0. If `pnpm lint` or `pnpm type-check` fail, fix the issues before proceeding.
  </action>
  
  <acceptance_criteria>

  - Workflow uses `setup-node@v4` with node-version 24 and pnpm cache
  - Workflow runs `pnpm lint` and `pnpm type-check`

  - `pnpm lint` exits 0
  - `pnpm type-check` exits 0
  </acceptance_criteria>
